## 1. 画面構成

左サイドバーから各画面に移動できます。

| メニュー | 内容 |
|----------|------|
| ダッシュボード | KPI・アラート・予定の一覧 |
| 見込み管理 | 契約前の案件パイプライン |

## 2. ダッシュボード

ログイン後に最初に表示される画面です。

### KPIカード（画面上部）

| カード | 表示内容 |
|--------|---------|
| 顧客数 | 登録されている顧客の総数 |
| 案件数 | 登録されている案件の総数 |
| 対応中の保守 | ステータスが「対応中」の保守対応件数 |
| 未入金アラート | 請求済みで未入金の件数と合計金額 |

各カードのリンクをクリックすると、該当の一覧画面に遷移します。

### 対応中の保守（左テーブル）

現在対応中の保守案件を最大8件表示します。

- **表示列**: 発電所名、顧客名、問合日、状況、状態
- 行をクリックすると保守対応の詳細画面を開きます

### 未入金アラート（右テーブル）

請求済みで入金が確認されていない案件を表示します。

- **表示列**: 案件名、顧客名、請求金額、入金予定日、状態
- 入金予定日を過ぎている場合、赤字で「超過」と表示されます
- 行をクリックすると請求詳細画面を開きます

### 請求予定（下部テーブル）

請求予定日が設定されているが、まだ請求していない案件を一覧表示します。

- **表示列**: 案件名、顧客名、請求予定日、請求方法、請求金額（税込）
- 請求予定日を過ぎている場合、赤字で「超過」と表示されます

## 3. 顧客管理

### 顧客一覧

全顧客を一覧表示します。

- **検索**: 顧客名・会社名・電話番号・メールで検索できます
- **表示列**: 顧客名、会社名、種別（法人/個人）、電話番号、メール、案件数
- 行をクリックすると顧客詳細画面を開きます

### 顧客の新規作成

1. 「＋ 新規顧客」ボタンをクリック
2. 以下の情報を入力:
   - **顧客名**（必須）
   - 会社名（入力すると自動的に「法人」になります）
   - 電話番号
   - メールアドレス
   - 郵便番号
   - 住所
3. 「保存」をクリック

### 顧客詳細画面

顧客の情報と、紐づく案件・添付ファイルを確認できます。

- **顧客情報**: 「編集」ボタンで情報を更新できます
- **案件一覧**: この顧客に紐づく案件の一覧。「＋ 案件を追加」で新規案件を作成できます
- **添付ファイル**: PDF・画像などのファイルを添付できます（詳細は[10. 添付ファイル](#10-添付ファイル)参照）

### 顧客の削除

顧客編集モーダルの下部にある「削除」ボタンから削除できます。
**注意**: 顧客を削除すると、紐づく案件・契約・請求データもすべて削除されます。

## 4. 案件管理

### 案件一覧

全案件を一覧表示します。

- **検索**: 案件名・案件番号・顧客名・設置住所・都道府県で検索できます
- **表示列**: 案件名、顧客名、都道府県、FIT期間、委託先、引渡日、監視システム

### 案件の新規作成

1. 「＋ 新規案件登録」ボタンをクリック
2. 顧客を選択（既存顧客 or 新規作成）
3. 案件情報を入力:
   - **案件名**（必須）
   - 案件番号、キー番号、都道府県、設置住所
   - 系統ID、系統認定日、FIT期間、需給開始日
   - お客さま番号、発電地点特定番号
   - 引渡日、販売会社、紹介者、販売価格
   - 監視システム、監視ID
   - 備考
4. 「保存」をクリック

### 案件詳細画面

案件詳細は4つのタブに分かれています。

#### 基本情報タブ

案件の基本情報をすべて表示します。「編集」ボタンで情報を更新できます。

表示される情報:
- 所在地情報（郵便番号、都道府県、住所、座標）
- パネル情報（kW、枚数、メーカー、型式）
- PCS情報（kW、台数、メーカー、型式）
- FIT・系統情報（系統ID、認定日、FIT期間、需給開始日）
- 監視情報（システム、ID、ユーザー名、PW、4G対応）
- その他（カギNo、自治会、旧所有者、販売会社、紹介者、各種日付、価格）

**契約情報**もこのタブに表示されます。契約がない場合は「＋ 契約情報を追加」ボタンが表示されます。

#### 保守対応タブ

この案件に紐づく保守対応の一覧を表示します。詳細は[5. 保守対応](#5-保守対応)を参照。

#### 定期保守タブ

この案件の定期保守記録を表示します。詳細は[6. 定期保守](#6-定期保守)を参照。

#### 請求タブ

この案件の契約情報と年度別の請求記録を表示します。詳細は[7. 請求管理](#7-請求管理)を参照。

## 5. 保守対応

発電所の故障・異常などへの対応を記録します。

### 保守対応一覧

- **検索**: 発電所名・顧客名・対象箇所・状況・管理番号で検索できます
- **フィルタ**: 「対応中」「完了」「すべて」のタブで絞り込めます（件数表示あり）
- **表示列**: 管理番号、発電所名、顧客名、問合日、発生日、対象箇所、状態

### 保守対応の新規作成

案件詳細画面の「保守対応」タブから追加できます。

1. 「＋ 保守対応を追加」をクリック
2. 以下の情報を入力:
   - 問合日
   - 発生日
   - 対象箇所
   - 状況（テキスト）
   - 対応（テキスト）
   - 報告（テキスト）
3. 「保存」をクリック

管理番号（例: 26001）は自動で採番されます。ステータスは「対応中」で作成されます。

### 保守対応の詳細・編集

保守対応の詳細画面では以下の操作ができます:

- **編集**: 「編集」ボタンで全項目を更新できます
- **完了にする**: 「完了にする」ボタンでステータスを「完了」に変更します
- **削除**: 案件詳細の保守対応タブから削除できます

### ステータスの流れ

```
対応中 → 完了
```

## 6. 定期保守

点検・除草・巡回などの定期的な保守作業を記録します。

### 定期保守の追加

案件詳細画面の「定期保守」タブから追加できます。

1. 「＋ 定期保守を追加」をクリック
2. 以下の情報を入力:
   - **記録日**（必須）
   - 作業種別: 点検 / 除草 / 巡回 / その他
   - 内容（テキスト）
3. 「保存」をクリック

### 定期保守の操作

- **追加**: 上記の通り
- **削除**: 各記録の削除ボタンで削除
- **編集**: 編集機能はありません。修正する場合は削除してから再度追加してください

## 7. 請求管理

### 請求の仕組み

請求データは以下の3層構造になっています:

```
案件 → 契約 → 年次記録（年度ごとの請求・入金）
```

請求機能を使うには、案件ごとに**契約情報**を作成し、その契約に対して**年次記録**を追加する必要があります。

### 契約情報の作成・編集

案件詳細の「請求」タブ、または「基本情報」タブから操作します。

契約の主な項目:

| 項目 | 説明 |
|------|------|
| 請求方法 | 「請求書」または「口座振替」 |
| 請求予定日 | 例: 12月1日 |
| 請求金額（税別/税込） | 年間の請求金額 |
| 年間保守料（税別/税込） | 保守契約の金額 |
| 土地賃料 | 月額の土地代 |
| 保険料 | 年間保険料 |
| その他 | その他費用 |
| 振替手数料 | 口座振替の場合の手数料 |
| 各種契約日 | 売買契約日、設備契約日、土地契約日、保守契約日 |
| 販売経路 | 販売店→ネオシス、ネオシス→紹介者、契約者名 |
| 保守委託 | 委託先、委託料、委託開始日 |
| 保守プラン | 点検・除草・駆付（各: なし / 年1回 / 無制限） |

### 年次記録の作成・編集

案件詳細の「請求」タブで年度ごとの請求・入金を管理します。

年次記録の主な項目:

| 項目 | 説明 |
|------|------|
| 年 | 対象年度（例: 2026） |
| 請求予定日 | 請求を送る予定の日付 |
| 請求日 | 実際に請求した日付 |
| 入金予定日 | 入金を見込む日付 |
| 入金日 | 実際に入金があった日付 |
| 請求明細 | 品目名と金額のリスト |
| ステータス | 未入金 / 請求済 / 入金済 |

### 請求一覧画面

請求一覧は自動的に以下の4セクションに分類されます:

#### 未入金アラート（赤枠）

請求済みで入金が確認されていない案件を表示します。

- 各行に入金日の入力欄があります
- 入金日を入力して「入金日を保存して消し込む」ボタンで一括保存できます
- 口座振替の場合は「振替」バッジが表示されます

#### 請求予定（青枠）

請求予定日が設定されているが、まだ請求していない案件です。

- 請求方法と保守プランが表示されます
- 予定日を過ぎている場合は「超過」と赤く表示されます

#### 口座振替（紫枠）

口座振替で振替待ちの案件を表示します。

#### 入金済（緑枠・折りたたみ）

入金確認済みの案件です。デフォルトでは折りたたまれています。

### 請求詳細画面

請求一覧で案件をクリックすると、詳細画面が開きます。

- **契約情報**: 請求方法、保守プランなどを確認・編集
- **今年度の請求**: 請求日・入金日・明細を入力。ステータスは日付の入力状況から自動判定されます
- **年度別請求履歴**: 過去年度の請求記録を確認・編集
- **保守情報**: 委託先・委託料などの保守関連情報
- **保守対応/定期保守**: この案件の保守履歴

### ステータスの自動判定

```
入金日あり     → 入金済
請求日あり     → 請求済
それ以外       → 未入金
```

### ダッシュボードにアラートを表示するために必要な項目

| アラート | 必要な設定 |
|---------|-----------|
| 未入金アラート | ① 契約を作成 ② 年次記録を作成 ③ 請求日を入力（入金日は空のまま） |
| 請求予定 | ① 契約を作成 ② 年次記録を作成 ③ 請求予定日を入力（請求日は空のまま） |
| 対応中の保守 | 保守対応を「対応中」ステータスで追加 |

## 8. 見込み管理

契約前の案件（見込み案件）を管理します。ローン申込から契約完了までの進捗をタスクベースで追跡できます。

### 見込み一覧

- **フィルタ**: 申込ステータスと契約ステータスの2軸で絞り込み
- カード形式で表示されます

### 見込み案件の新規作成

1. 「＋ 案件を追加」をクリック
2. 以下の情報を入力:
   - **顧客名**（必須）
   - **物件名**（必須）
   - ローン会社（アプラス / ジャックス / 空欄）
   - 物件所在地、設備費、土地費、融資額、kW数、紹介元、商談開始日
3. 「保存」をクリック

### 見込み詳細画面

#### KPIカード

設備費・土地費・合計・融資額を表示します。

#### 基本情報

日付・住所フィールドを直接編集できます（自動保存）:

- 商談開始日
- 申込提出日
- 審査結果日
- 売買契約日
- 土地契約日
- 引渡し予定日
- 物件所在地

#### タスクパネル（申込 / 契約）

ローン会社に応じたチェックリストが表示されます。

**申込タスク（例: アプラス）:**
- ローン申込書
  - └ 金利変動確認欄
- 確認書（アンケート）
- 確認票（保険）
- 収入証明書

**契約タスク:**
- 売買契約書
  - └ 顧客へ郵送
  - └ 代理店へ郵送
- 土地契約書
  - └ 顧客へ郵送
  - └ 代理店へ郵送
- 住民票
- 印鑑証明書
- 契約チェック

各タスクのチェックボックスをクリックすると自動保存されます。メモ欄も自動保存です。

#### ステータス変更

申込・契約それぞれのパネルで、ステータスボタンをクリックして変更します。

**申込ステータス:**
```
未 → 提出済 → 通過 / 不通 / 不可
```

**契約ステータス:**
```
未 → 完了 / 不可
```

### 顧客管理への登録（コンバージョン）

契約ステータスが「完了」になると、「顧客管理へ登録 →」ボタンが表示されます。

1. ボタンをクリック
2. 確認ダイアログで内容を確認
3. 「登録する」をクリック

見込み案件の情報をもとに、顧客・案件が自動作成されます。登録後は「✓ 顧客管理済」と表示されます。

## 9. CSVインポート

CSVファイルから案件データを一括で取り込みます。

### 対応フォーマット

| フォーマット | 説明 |
|-------------|------|
| テンプレート形式 | アプリからダウンロードしたテンプレートに合わせた形式 |
| 既存データ形式 | 2行ヘッダーの既存管理データ |
| 全体シート形式 | エイジフル低圧一覧の全体シート（契約・請求情報含む） |

フォーマットは自動判定されます。

### インポート手順

1. **テンプレートのダウンロード**（任意）: 「CSVテンプレートをダウンロード」ボタンをクリック
2. **CSVファイルの選択**: ファイルをドラッグ&ドロップ、またはクリックして選択
3. **プレビュー確認**: 取込対象のデータが一覧で表示されます。内容を確認してください
4. **インポート実行**: 「○件をインポート」ボタンをクリック
5. **結果確認**: 成功件数・失敗件数が表示されます。失敗した行はエラー詳細を確認してください

### 文字コード

UTF-8（BOM付き/なし）とShift-JISに対応しています。自動判定されます。

### 顧客の名寄せ

同じ顧客名の案件は、自動的に同一顧客として紐づけられます。1人の顧客に複数の案件が登録されます。

### インポート前のデータクリア

既存データを全て消してから再インポートする場合は、SupabaseのSQLエディタで以下を実行してください:

```sql
TRUNCATE customers CASCADE;
```

これにより顧客・案件・契約・請求・保守データが全て削除されます。

## 10. 添付ファイル

顧客に対してファイルを添付できます。

### 対応ファイル形式

PDF、PNG、JPG、JPEG、GIF、WEBP、DOC、DOCX、XLS、XLSX

### ファイルの追加

1. 顧客詳細画面の「添付ファイル」セクションで「＋ ファイルを追加」をクリック
2. ファイルをドラッグ&ドロップ、またはクリックして選択
3. 説明（任意）を入力
4. 「アップロード」をクリック

### ファイルの操作

- **開く**: 「開く」ボタンでファイルを新しいタブで表示
- **削除**: 削除ボタン（🗑）をクリック → 確認ダイアログで「削除」

## 付録: データの関係図

```
顧客 (customers)
 ├── 案件 (projects)  ※1顧客に複数案件
 │    ├── 契約 (contracts)  ※1案件に1契約
 │    │    └── 年次記録 (annual_records)  ※年度ごと
 │    ├── 保守対応 (maintenance_responses)  ※複数
 │    └── 定期保守 (periodic_maintenance)  ※複数
 └── 添付ファイル (attachments)  ※複数

見込み案件 (prospects)  ※独立、契約完了後に顧客・案件に変換
```
